模式 Pattern

设计模式

设计模式是可用于某类特定问题的实践方法。

“没有必要重复发明轮子”，iUAP Design 设计模式是用友多年toB经验的积累，我们提取中后台业务中久经考验的有效的设计方案,将其应用于类似问题的解决.将设计理论形式化,记录最好的实践工作。 成熟的模式更容易解决常见问题，正确的模式能帮用户熟悉界面、提高效率。

模式的优势

  • 节省新项目的时间和精力.
  • 提高设计方案的质量.
  • 促进设计师与程序员的沟通.
  • 帮助设计师成长.

交互设计模式不仅仅涉及结构和元素组织,同时还关注响应用户活动的动态行为与变化.

模式的层次

  • 定位模式：应用于概念层面，帮助界定产品对于用户的整体定位，对应概念原则。

  • 结构模式：解答如何在屏幕上安排信息和功能元素等问题，对应界面原则。

  • 行为模式：解决具体的交互问题，对应行为原则。
